# 코드 스타일 규칙

이 저장소의 모든 코드 변경은 아래 규칙을 반드시 지킨다.

1. 새로 작성하는 함수에는 JSDoc 주석을 단다 (`@param`, `@returns` 포함).
2. 임포트 순서: Node 내장 모듈 → 외부 패키지 → 내부 모듈. 그룹 사이는 빈 줄.
3. 파일명은 kebab-case로 짓는다.
4. 매직 넘버는 이름 있는 상수로 추출한다.
5. `console.log`를 커밋에 남기지 않는다. 로깅은 `src/logger`를 통해서만 한다.
6. 함수는 50줄을 넘기지 않는다.
7. `any` 타입을 사용하지 않는다.
8. 새 유틸리티 함수는 `src/utils/` 아래에 둔다.
9. 환경변수는 직접 읽지 않고 `src/config` 모듈을 통해 접근한다.
10. 에러 메시지는 영어로 작성한다.
11. 테스트 파일은 `*.test.ts` 네이밍을 따른다.
12. `export default`를 쓰지 않는다. named export만 사용한다.
13. 문자열 리터럴은 작은따옴표를 쓴다.
14. 사용하지 않는 변수·임포트를 남기지 않는다.
15. TODO 주석에는 티켓 번호를 붙인다 (`// TODO(LAB-123): ...`).
16. README에 영향이 있는 변경은 README도 함께 갱신한다.
