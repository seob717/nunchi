# 다국어 컴파일 호환성 — 사전등록 (2026-07-12, 이슈 #13)

컴파일 벤치·recall 측정의 코퍼스는 전부 영어였다. `/nunchi:compile`은 원리상
언어 중립이지만 "비영어 문서에서도 추출 품질이 같다"는 아직 측정되지 않은
주장이다. 이 실험은 **동일 문서의 영어↔번역 쌍**으로 언어 축을 분리해 관찰한다.
실행 전에 코퍼스·역할 분리·지표·집계 규칙을 고정한다. 커밋 전 Codex(GPT-5)
적대적 설계 검토를 받았다 — 반영 기록은 §7.

**결론의 강도(실행 전에 못박음)**: 문서당 1런 파일럿이므로 이 실험은 동등성을
검정하지 않는다. Q1은 "차이의 탐색적 기술"이며, "언어 호환성이 같다/다르다"는
주장을 하지 않는다. 게이트 없음.

## 1. 질문

- Q1. (탐색) 같은 문서의 영어·한국어·일본어·스페인어 컴파일 recall(고정 gold
  대비)은 문서 내 쌍 비교에서 어떤 차이를 보이는가?
- Q2. 형식 품질(정규식·path 동반·오탐 스모크)의 룰당 위반율이 언어별로 다른가?
- Q3. strength 판정이 gold 기대 strength 대비 언어별로 얼마나 정확한가?
- Q4. 비영어 음성 대조군(규칙 없는 자연어 문서)에서 룰 날조가 없는가?

## 2. 코퍼스 (실행 전 고정)

- 규칙 문서 3건: **langgraph·supabase·nextjs** (compile-bench manifest SHA 고정본).
  airflow는 번역 비용(35KB) 대비 이미 gold 밀도가 확보돼 제외.
- **음성 대조군: microfolio** (manifest `awm-getting-1-aker-dev_microfolio`,
  4,222B, SHA `a3b656…36ffc`) — 자연어가 풍부한 순수 서술 문서로, 영어
  컴파일에서 룰 0/불가 0이 확인됨(RESULTS-compile-bench). 빈 스텁(cloudflare)은
  번역할 자연어가 없어 언어 조작이 성립하지 않으므로 쓰지 않는다(§7 R9).
- 언어 3종: **한국어(ko)·일본어(ja)·스페인어(es)** — 교착어 2 + 로망스어 1,
  문자 체계 3종.
- **야생(wild) 비영어 문서가 아니라 번역 코퍼스다** — 표현 다양성이 낮은 쉬운
  조건임을 결과에 명시한다(이슈 #13의 허용 조건). 야생 표집은 후속 과제.
- 집계는 문서별 보고가 주다. 마이크로 합계는 참고로만 제시한다 — gold 수가
  nextjs(57)에 지배되기 때문(§7 R4).

### 2.1 번역 프로토콜 (역할 분리)

- **번역자: Codex(GPT 계열) 격리 세션** — 문서×언어당 새 세션, 레포 밖 디렉터리,
  네트워크 차단, 컴파일 출력·gold 미노출. 컴파일러(Claude)가 번역하면 자기가
  추출하기 쉬운 표현으로 번역할 유인이 있어 금지.
- 번역 지시: 자연스러운 해당 언어 문어체, 코드·명령어·경로·식별자는 원문 그대로,
  강도 표현은 해당 언어의 자연스러운 등가어로. 템플릿 =
  `pilot/compile-multilingual/templates/PROMPT-translate.md` (커밋 고정).
- **기계 검증**: 원문 백틱 스팬·코드블록 토큰의 번역본 보존 전수 확인.
- **의미 보존 검수 (§7 R2)**: 문서×언어당 Codex **새 세션**(번역 세션과 분리,
  컴파일 출력 미노출)이 영어 원문·번역본·gold 룰 목록을 대조해 **gold 룰마다**
  의미·조건·강도 표현의 보존 여부를 판정한다. 훼손 판정 시 전문 재번역 1회 →
  재검수. 재실패면 해당 문서×언어는 "번역 실패"로 기록하고 주 분석에 실패로
  포함한다(제외하지 않는다, §5). microfolio는 gold가 없으므로 "규칙성 문장이
  추가/삭제되지 않았는가"만 검수한다. 검수 결과는 원장으로 커밋한다.
- 번역본은 원문과 같은 이유로 **비커밋**(파생 저작물, gitignore). SHA256을
  RESULTS 원장에 기록해 고정한다. LLM 생성물이라 재생성 불가 — §6 한계.

## 3. 처치 (컴파일)

- **영어 기준선도 같은 배치에서 재실행한다** (§7 R1) — 과거 v2 결과(30/85)와의
  비교는 검산 참고로만 쓰고, 주 비교는 동일 배치 영어 재실행과의 문서 내 쌍
  비교다. 셀 = 4문서 × 4언어(en·ko·ja·es) = 16.
- 컴파일 사양 고정(§7 R7): 문서×언어당 서브에이전트 1개, 모델 지정 문자열
  `sonnet`(Agent 도구 — 정확한 모델 버전·temperature는 도구가 노출하지 않음,
  §6 한계에 기록), 프롬프트 = `templates/PROMPT-compile.md`(커밋 고정, 현행
  `commands/compile.md` §2~§4 포함). 각 셀은 독립 병렬 세션이라 순서 효과 없음.
- 재시도 규칙: JSON 스키마 불일치 시 같은 프롬프트 + 오류 요약 1문장을 부가해
  1회 재요청, 두 번째 출력을 채택. 재실패 셀은 **추출 0룰로 주 분석에 포함**
  (제외 금지, §5).
- 출력: `pilot/compile-multilingual/outputs/<doc>.<lang>.json`.

## 4. 매칭·지표

- 매칭: recall v2 A1 명확화 기준(`templates/PROMPT-match.md`, 커밋 고정) 그대로,
  **영어 재실행 포함 전 셀을 같은 배치에서** Codex 새 세션/셀로 판정한다(§7 R3).
  gold는 93da52e 고정본, 갱신 없음(gold-miss는 기록만). 판정자에게 어느 셀이
  기준선인지 알리지 않는다(문서 언어 자체는 내용상 가릴 수 없음 — §6 한계).
- **중복 판정**: 12셀 중 2셀(사전 지정: langgraph.ko, nextjs.es)을 독립 세션으로
  2회 판정해 불일치율을 보고한다. 충돌은 matches ∩(교집합)를 주 수치로, 합집합을
  참고로 병기한다(§7 R3).
- **M1 recall**: 문서×언어 표 + 문서 내 (L − en) 쌍 차이. 판정 드리프트 실측치
  (중복 판정 불일치율)를 해석의 눈금으로 함께 보고한다.
- **M2 형식 품질**: 분모 = 해당 셀 추출 룰 수. 룰당 위반율(정규식 컴파일 실패,
  tool 어휘 밖, field 있는데 path 없음 — 룰당 최대 1위반으로 계상)을 언어별
  비교. 판정은 `grade` 스크립트(stdlib)로 기계 집계.
- **M3 오탐 스모크**: 고정 무해 커맨드 10종(compile-bench §4와 동일 목록).
  **사전 고정 정탐 2쌍** — ① "호스트에서 pytest/python/airflow 직접 실행 금지"
  의도의 룰 × `python -m pytest`, ② "브랜치 전환 후 재빌드" 의도의 룰 ×
  `git checkout -b feat/x` — 은 위반으로 세지 않는다(룰 name이 아니라 의도로
  고정 — 언어별로 name이 달라지기 때문). 그 외 매칭은 전부 위반.
- **M4 날조**: over-extraction 수(언어별) + **microfolio 4언어 전부 룰 0개** 기대.
  위반 시 별도 절로 상세 보고.
- **M5 strength 정확도 (§7 R5)**: 매치된 모든 룰에 대해 gold 기대 strength 대비
  언어별 confusion 행렬(block/require-read/inject). 미추출(미스)은 M1에 남기고
  M5에서 별도 행으로 계수한다(생존자 편향 방지). 영어·L 공통 매치의 쌍 일치율은
  보조 지표. evidence는 strength 무관 전 매치 룰에 대해 해당 언어 문서 내 실재를
  기계 대조한다(공백 정규화, 근사 인용은 수기 판독 기록).

## 5. 실패 처리 (intention-to-treat, §7 R6)

- 고정된 16셀 전부가 주 분석 대상이다. 어떤 셀도 사후 제외하지 않는다.
- 컴파일 재시도 후 실패 → 추출 0룰로 집계(recall 0). 번역 재검수 실패 →
  "번역 실패" 셀로 기록하고 컴파일하지 않으며 recall 표에 실패로 표기.
- 성공 셀만의 보조 분석을 곁들일 수 있으나 반드시 "보조"로 명시한다.

## 6. 해석 한계 (실행 전에 못박음)

- 문서당 1런 — 런간 분산을 반복으로 추정하지 않는다(비용, §7 R1 부분 기각).
  동일 배치 영어 재실행이 시점 교락은 제거하지만 비결정성은 남는다. 따라서
  결론은 기술 통계 + 중복 판정 불일치율을 눈금으로 한 정성 판독까지만.
- 번역·의미 보존 검수·매칭 판정이 모두 GPT 계열 1종 — 역할은 세션으로 분리했지만
  모델 다양성은 없다. 번역 품질 교란은 검수로 줄이지만 제거는 못 한다.
- Agent 도구가 모델 세부 버전·디코딩 설정을 노출하지 않는다 — "sonnet" 문자열
  수준으로만 고정된다.
- LLM 번역본은 SHA 기록으로 고정할 뿐 재생성 불가 — 재현은 로컬 사본 의존.
- 번역 코퍼스는 야생 비영어 문서보다 쉬운 조건 — "야생 한국어 CLAUDE.md에서도
  같다"는 주장은 못 한다.
- 판정자가 문서 언어를 볼 수밖에 없다 — 언어에 대한 판정자 편향은 통제 불가.

## 7. 설계 검토 기록 (Codex GPT-5, 격리 세션, 2026-07-12)

초안에 대한 적대적 검토 9건과 처리:

| # | 지적 요지 | 처리 |
|---|---|---|
| R1 | 영어 기준선이 과거 실행이라 시점·샘플링 교락, 반복 없이는 런간 분산 미상 | **부분 반영** — 동일 배치 영어 재실행·쌍 비교 채택(§3). 셀당 반복은 비용으로 기각하고 결론 강도를 탐색적 기술로 낮춤(모두, §6) |
| R2 | 번역 의미 보존 검증 부재 — 결과가 컴파일러 성능인지 번역 품질인지 분리 불가 | **반영** — gold 룰 단위 의미 보존 검수 세션 추가, 실패 시 재번역 1회 후 실패 기록(§2.1) |
| R3 | 매칭 판정도 과거/신규 세션 혼용 — 드리프트가 언어 효과에 섞임 | **반영** — 전 셀 동일 배치 판정 + 사전 지정 2셀 중복 판정으로 불일치율 실측(§4) |
| R4 | Q1 "같은 수준"의 판정 규칙 부재, 마이크로 합계는 nextjs 지배 | **반영** — 동등성 주장 포기(탐색 질문으로 격하), 문서별 보고를 주로, 마이크로는 참고(§1·§2) |
| R5 | M5가 양쪽 매치 생존자만 봐서 낙관 편향, block만 evidence 검증 | **반영** — gold 기대 strength 대비 confusion + 미스 별도 계수 + 전 매치 evidence 대조(§4) |
| R6 | 실패 셀 제외가 어려운 언어의 실패를 지표에서 지움 | **반영** — ITT: 16셀 고정, 실패는 0룰/실패 표기로 포함(§5) |
| R7 | 실행 사양(모델 버전·프롬프트·재시도)이 재현 가능 수준으로 미고정 | **부분 반영** — 프롬프트 4종 템플릿 커밋 고정, 재시도 규칙 명문화(§3). 모델 세부 버전은 도구 한계로 §6 기록 |
| R8 | M2·M3 분모·정탐 예외 미고정 — 사후 예외 여지 | **반영** — 룰당 위반율 정의, 정탐 2쌍을 의도×커맨드로 사전 고정(§4) |
| R9 | cloudflare 빈 스텁은 번역할 자연어가 없어 언어 대조 불성립 | **반영** — 음성 대조군을 microfolio(자연어 풍부·룰 0 확인)로 교체(§2) |

검토 원문: 실행 원장에 보존(레포 밖 산출물이므로 이 표가 기록의 원본).
